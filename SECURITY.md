# Security Policy

## Supported Versions

We currently support the following versions of **HieroBlockBridge** with security updates:

| Version | Supported          |
| ------- | ------------------ |
| v0.1.x  | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

We take the security of HieroBlockBridge and its users seriously. If you discover a vulnerability, we would like to know about it so we can take steps to address it as quickly as possible.

Please do **not** report security vulnerabilities via public GitHub issues.

Instead, please report them by emailing: **kabufredrick9@gmail.com**

You should receive a response within 48 hours. If for some reason you do not, please follow up via email to ensure we received your original message. 

### What to include in your report

Please provide as much information as possible, including:
- A detailed description of the vulnerability and its potential impact
- The versions of HieroBlockBridge affected
- Steps to reproduce the vulnerability (a proof-of-concept script/repo is ideal)
- Any potential workarounds or mitigations

### Remediation Process

1. **Acknowledge:** We will acknowledge receipt of your report within 48 hours.
2. **Verify:** We will work with you to understand and verify the vulnerability.
3. **Patch:** We will develop a patch and request your feedback/testing.
4. **Publish:** Once verified, an advisory will be published along with a new release containing the fix.

We will credit you in the release notes and advisory for responsible disclosure (unless you prefer to remain anonymous).
